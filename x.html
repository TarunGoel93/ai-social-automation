01 · features

Everything you need to post on autopilot

Six tightly integrated capabilities, zero third-party auth SDKs.

🤖
AI Mode
Describe your idea and let the AI craft the perfect post — captions, tone, hashtags, all of it. Just a prompt and a platform.
✍️
Manual Mode
Drag-and-drop your image, write your caption, and publish directly. Full control, zero abstraction.
🕐
Scheduled Queue
Queue posts for any future date & time. A background daemon polls every 30 s and fires them via the n8n webhook.
📋
Queue Dashboard
Live panel showing pending, posted, and failed posts. Cancel or manually fire any queued item on demand.
🔗
n8n Integration
Webhook-driven delivery to LinkedIn and Instagram through importable n8n workflows — no platform SDK headaches.
💾
Persistent Storage
Every queued post — including uploaded images — is stored in SQLite so nothing is lost across server restarts.

04 · quick start

Up and running in 4 steps

01
Clone the repository
Pull the code from GitHub and change into the project directory.
bash 
git clone https://github.com/TarunGoel93/ai-social-automation.git
cd ai-social-automation
02
Install Python dependencies
Three lightweight packages — Flask, CORS support, and Requests.
bash 
pip install -r requirements.txt
03
Set your n8n webhook URL
Point the app at your n8n production webhook. The default test URL only works while the workflow is open in n8n.
bash 
export N8N_WEBHOOK_URL="https://<your-n8n-instance>/webhook/<your-path>"
⚠️
Production note: Replace /webhook-test/ with /webhook/ and activate the workflow before going live.
04
Start the server
Flask starts on port 5000. The background scheduler launches automatically in a daemon thread.
bash 
python app.py
# → Running on http://localhost:5000
✅
Open http://localhost:5000 — the dashboard is live. The SQLite database is created automatically on first run.

06 · api reference

All six endpoints

RESTful, JSON-first, with multipart support for image uploads.

Method Route Description
POST /submit Immediate or scheduled single post (AI or manual)
POST /queue Add directly to persistent queue — scheduled_time required
GET /posts List all posts · filter via ?status=pending|posted|failed
DEL /posts/:id Cancel and delete a pending post (removes image from disk too)
POST /posts/:id/fire Manually trigger a pending post immediately
GET /health Check whether the n8n webhook is reachable
Example — AI mode payload
json 
{
  "mode":           "ai",
  "prompt":         "Announce our new product launch with excitement",
  "tone":           "professional",
  "platforms":      "linkedin",
  "scheduled_time": "2025-12-01T10:00:00"  // omit for immediate
}
07 · database schema

posts table

Single SQLite table. Auto-created on first run via init_db().

idINTEGER PK
Auto-increment primary key
modeTEXT NOT NULL
ai or manual
platformsTEXT NOT NULL
instagram · linkedin · both
captionTEXT
Manual mode caption text
image_pathTEXT
Local path to stored image (never exposed via API)
image_filenameTEXT
Original sanitized filename
image_mimetypeTEXT
e.g. image/jpeg
promptTEXT
AI mode post idea
toneTEXT
AI mode tone hint (optional)
scheduled_timeTEXT NOT NULL
ISO-8601 datetime string
statusTEXT DEFAULT pending
pending · posted · failed
created_atTEXT
Auto set to datetime('now')
fired_atTEXT
Timestamp when scheduler fired the post
error_msgTEXT
Webhook error detail when status=failed
